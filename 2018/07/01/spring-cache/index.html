缓存可以存储经常使用的信息， 这样每次需要的时候，这些信息都是立即可用的。

启用对缓存的支持

Spring3.1 内置了五个缓存管理器实现

  • SimpleCacheManager
  • NoOpCacheManager
  • ConcurrentMapCacheManager
  • CompositeCacheManager
  • EhCacheCacheManager

Spring Data提供了两个缓存管理器

  • RedisCacheManager
  • GemfireCacheManager

在一个配置类上添加@EnableCaching，这样就可以启动注解驱动的缓存。@EnableCaching会创建一个切面（aspect）并触发缓存注解的（pointcut）。根据缓存所使用的注解以及缓存的状态，这个切面会从缓存中获取数据，并将数据添加到缓存中或者从缓存中移除某个值。

代码1不仅启用了注解驱动的缓存，还声明了一个缓存管理器（cache manager）的bean。CacheManager是Spring缓存抽象的核心， 它能够和多个流行的缓存进行集成。本次集成EhCacheCacheManager， Spring提供了EhCacheManagerFactoryBean来生成Ehcache的CacheManager，然后被注入Spring Ehcache的EhCacheCacheManager之中。EhCacheManagerFactoryBean是一个工厂bean，所以注册在Spring应用上下文中的并不是EhCacheManagerFactoryBean的示例，而是CacheManager的示例，因而适合注入EhCacheCacheManager中。

为方法添加注解以支持缓存

表1 Spring 提供了四个注解声明缓存规则

注解 描述
@Cacheable Spring在方法调用之前从缓存中查找返回值，如果能够找到就返回缓存的值，否则这个方法就被调用
@CachePut Spring在方法调用前不会检查缓存， 始终会被调用并缓存返回值
@CacheEvict Spring清除缓存中指定的一个或多个条目
@Caching 这是个分组的注解，能同时应用多个其他缓存的注解

填充缓存

表1知道@Cacheable和@CachePut可以填充缓存。

表2 @Cacheable和@CachePut的共有属性

属性 类型 描述
value String[] 要使用的缓存名称
condition String SpEL表达式， 如果得到的值是false，不会将缓存应用到方法调用上
key String SpEL表达式，用来计算自定义的缓存的key
unless String SpEL表达式，如果得到的值是true的话， 返回值不会放到缓存中

unless属性只能阻止将对象放入缓存中，但是这个方法调用的时候依然会从缓存中查找，如果找到匹配的值就会返回找到的值。与之不同，如果condition的表达式计算结果为false， 方法调用的过程中缓存是禁用的，即不会从缓存中查找，返回值也不会放入缓存中。

表3 Spring提供了多个用来定义缓存规则的SpEL扩展

表达式 描述
#root.args 传递给缓存方法的参数，形式为数组
#root.caches 该方法执行时对应的缓存， 形式为数组
#root.target 目标对象
#root.targetClass 目标对象的类， 是#root.target.class的简写形式
#root.method 缓存方法
#root.methodName 缓存方法的名字
#result 方法定义的放回值， 不能用在@Cacheable注解上
#Argument 任意方法的参数名（如#argName)或参数索引（如#a0或#p0）

表4 @CacheEvict注解的属性， 指定了那些缓存条目应该被移除掉

属性 类型 描述
value String[] 用使用的缓存名称
key String SpEL表达式，用来计算自定义的缓存key
condition String SpEL表达式，false：缓存不会被应用到方法调用上
allEntries boolean true：特定缓存的所有条目都会被移除
beforeInvocation boolean true，方法调用之前移除条目，false：方法调用之后移除条目
