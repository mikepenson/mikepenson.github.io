接口定义（Hello.idl)

创建CORBA应用程序的第一步是使用OMG的接口定义语言（IDL）指定所有对象及其接口。IDL具有类似于C ++的语法，可用于定义模块，接口，数据结构等。IDL可以映射到各种编程语言。Java的IDL映射在IDL中汇总为Java语言映射摘要。

以下代码用OMG IDL编写，描述了一个CORBA对象，其sayHello（）操作返回一个字符串，其shutdown（）方法关闭ORB。要了解有关OMG IDL语法和语义的更多信息，请阅读CORBA 2.3.1规范的第3章。

Hello.idl

注意：

用OMG IDL中编写代码时，请勿使用接口名称作为模块的名称。这样做可能会在使用来自不同供应商的工具进行编译时产生不一致的结果，从而危及代码的可移植性。例如，可以使用Sun Microsystems的IDL到Java编译器编译包含相同名称的代码，并获得一个结果。使用其他供应商的IDL编译到Java编译器的相同代码可能会产生不同的结果。

要完成应用程序，只需提供server（HelloServer.java）和client（HelloClient.java）实现。

服务端实现 (HelloServer.java)

示例服由两个类组成，即servant和server。servant，HelloImpl，是Hello IDL接口的实现; 每个Hello实例都由HelloImpl实例实现。servant是HelloPOA的子类，它由 idlj编译器从示例IDL生成。servant为每个IDL操作包含一个方法，在本例中为 sayHello（）和shutdown（）方法。Servant方法就像普通的Java方法一样; 处理ORB的额外代码，包含编组参数和结果等，由骨架提供。

在HelloServer类有服务端main（） 方法，其中：

  • 创建并初始化ORB实例
  • 获取对根POA的引用并激活 POAManager
  • 创建一个servant实例（一个CORBA Hello对象的实现 ）并告诉ORB它
  • 获取命名上下文的CORBA对象引用，在该命名上下文中注册新的CORBA对象
  • 获取根命名上下文
  • 在名称“Hello”下的命名上下文中注册新对象
  • 等待从客户端调用新对象

此示例提供了一个临时对象服务器的示例。有关具有持久对象服务器的“Hello World”程序的示例，请参见示例2：具有持久状态的Hello World。有关CORBA服务器的更多讨论，请参阅开发服务器。

有关代码的更多讨论，请参阅详细的教程主题 Java IDL入门：开发Hello World Server。

构建和运行Hello World

尽管设计简单，Hello World程序允许您学习和试验开发几乎所有使用静态调用的 CORBA程序所需的所有任务。在编译时已知对象的接口时，将使用静态调用，该调用使用客户端存根进行调用，并使用服务器框架来调用正在调用的服务。如果在编译时未知接口，则必须使用动态调用。

此示例需要命名服务，该服务是一种CORBA服务，它允许通过将名称绑定到对象引用来命名CORBA对象。的名称绑定可被存储在命名服务，而客户端可以提供名称，以获得所期望的对象的参考。此版本的Java SE附带的命名服务的两个选项包括orbd，一个包含一个Bootstrap服务、一个瞬态命名服务，一个持久命名服务和一个服务器管理器的守护进程，以及 一个为其提供向后兼容的临时命名服务tnameserv。此示例使用orbd。

运行此示例时，请记住，在使用Solaris软件时，必须成为root用户才能在1024以下的端口上启动进程。因此，建议您使用大于或等于1024的端口号。 -ORBInitialPort选项用于覆盖此示例中的默认端口号。以下说明假定您可以将端口1050用于Java IDL对象请求代理守护程序orbd。如有必要，您可以替换其他端口。在Windows计算机上运行这些示例时，请在路径名中添加反斜杠（\）。

要在开发计算机上运行此客户端 - 服务器应用程序：

  1. 切换到包含文件Hello.idl的目录 。

  2. 在IDL文件上运行IDL-to-Java编译器idlj以创建存根和骨架。此步骤假定您已在路径中包含java / bin目录的路径。

    必须将-fall选项与idlj 编译器一起使用才能生成客户端和服务器端绑定。此命令行将生成默认的服务器端绑定，该绑定采用POA继承服务器端模型。有关idlj选项的更多信息，请参阅IDL-to-Java编译器选项。

    该IDLJ编译器会生成大量的文件。生成的实际文件数取决于编译IDL文件时选择的选项。生成的文件提供标准功能，因此您可以忽略它们，直到部署和运行程序为止。idlj 编译器使用-fall命令行选项为Hello.idl生成的文件：

    • HelloPOA.java

      此抽象类是基于流的服务器框架，为服务器提供基本的CORBA功能。它扩展了 org.omg.PortableServer.Servant，并实现了InvokeHandler接口和HelloOperations接口。服务器类 HelloImpl扩展了HelloPOA。

    • _HelloStub.java

      此类是客户端存根，为客户端提供CORBA功能。它扩展了 org.omg.CORBA.portable.ObjectImpl并实现了 Hello.java接口。

    • Hello.java

      该接口包含我们的IDL接口的Java版本。所述Hello.java界面延伸 的org.omg.CORBA.Object，提供标准的CORBA对象的功能性。它还扩展了HelloOperations 接口和org.omg.CORBA.portable.IDLEntity。

    • HelloHelper.java

      此类提供辅助功能，特别是将CORBA 对象引用强制转换为正确类型所需的 narrow（）方法.Helper类负责读取和写入CORBA流的数据类型，以及从Any 中插入和提取数据类型。Holder类委托Helper类中的方法进行读写。

    • HelloHolder.java

      最后一个类包含Hello类型的公共实例成员 。只要IDL类型是out或 inout参数，就使用Holder类。它为org.omg.CORBA.portable.OutputStream和 org.omg.CORBA.portable.InputStream参数提供了操作，CORBA允许这些参数，但它们不容易映射到Java的语义。Holder类委托Helper类中的方法进行读写。它实现了org.omg.CORBA.portable.Streamable。

    • HelloOperations.java

      该接口包含sayHello（）和 shutdown（）方法。IDL-to-Java映射将IDL接口上定义的所有操作放入此文件中，该文件由存根和骨架共享。
